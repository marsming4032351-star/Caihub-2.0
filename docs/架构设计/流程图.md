# CaiHub 可视化流程图

本文档使用 Mermaid 描述 CaiHub 当前阶段的核心流程，GitHub 仓库页面可直接渲染。

## 1. 业务主流程

```mermaid
flowchart TD
    A[菜品主数据 Dish] --> B[菜品标准 Dish Standard]
    B --> C[提交出品事件 Production Event]
    C --> D[读取标准]
    C --> E[接收识别结果]
    D --> F[质量规则服务 Quality Rule Service]
    E --> F
    F --> G[生成质量裁决]
    G --> H[生产事件落库]
    G --> I[生成质量领域事件]
    H --> J[后续分析 / 训练 / 治理]
    I --> J
```

## 2. 后端调用链

```mermaid
flowchart LR
    Client[客户端 / 前端 / 平台调用方]
    API[FastAPI Endpoints]
    Service[Services]
    Repo[Repositories]
    DB[(PostgreSQL / SQLite)]

    Client --> API
    API --> Service
    Service --> Repo
    Repo --> DB
```

## 3. 出品事件裁决链路

```mermaid
sequenceDiagram
    participant U as 调用方
    participant P as Production API
    participant S as ProductionEventService
    participant DS as DishStandardService
    participant Q as QualityRuleService
    participant R as ProductionEventRepository
    participant DB as Database

    U->>P: POST /api/v1/production/events
    P->>S: create_event(payload)
    S->>DS: get_standard_by_dish_key(dish_id)
    DS-->>S: 标准 / 默认标准
    S->>Q: evaluate(capture, recognition, standard)
    Q-->>S: quality_result
    S->>R: create(production_event)
    R->>DB: INSERT production_events
    DB-->>R: created row
    R-->>S: created event
    S-->>P: ProductionEventRead + quality event
    P-->>U: 201 Created
```
