# CaiHub AI Company 业务版项目说明

本文档面向业务合作方、管理层和投资人，用于说明 CaiHub 最终想成为一家什么样的公司，以及当前仓库在这条路线上的位置。

## 1. CaiHub 是什么

CaiHub 不是一个单纯“识别菜品图片”的工具。

CaiHub 的定义是：

```text
餐饮行业的 AI 运营系统
```

它的核心目标是：

```text
用 AI Agent 复刻餐厅运营体系，让餐饮经验变成数据资产。
```

这意味着 CaiHub 做的不是单点识别、单点巡检或单点报表，而是要把餐饮组织中的研发、出品、运营、营销和数据沉淀能力，变成一个可持续进化的 AI Company。

## 2. 组织结构

```text
Mars（Founder）
  │
  ▼
CaiHub CEO Agent
  │
  ├── Vision QA Agent（CaiBox）
  ├── Menu R&D Agent
  ├── Store Ops Agent
  ├── Marketing Agent
  └── Data Asset Agent
```

### 角色说明

#### Mars（Founder）
- 负责战略方向、商业模式、组织边界与最终决策。

#### CaiHub CEO Agent
- 负责目标拆解、任务协同、结果汇总和组织级决策支持。

#### Vision QA Agent
- 负责视觉质检、异常识别、多模态输入采集和质量判断辅助。

#### Menu R&D Agent
- 负责菜单研发、菜品标准沉淀、迭代建议和知识积累。

#### Store Ops Agent
- 负责门店执行、SOP 稳定性、出品波动分析和运营优化。

#### Marketing Agent
- 负责活动、增长、内容和品牌传播策略。

#### Data Asset Agent
- 负责把所有运营过程沉淀为结构化数据资产与可复用知识。

## 3. 技术架构

### Layer 1 — Hardware Layer
设备层：

- CaiBox
- 摄像头
- 照度系统
- 温度传感器

职责：采集真实餐饮现场的数据输入。

### Layer 2 — Vision Layer
视觉层：

- 色泽识别
- 摆盘识别
- 食材识别
- 比例识别

职责：把图像和环境信号转化为结构化事实。

### Layer 3 — Agent Layer
Agent 层：

- CEO Agent
- Vision QA Agent
- Menu R&D Agent
- Store Ops Agent
- Marketing Agent
- Data Asset Agent

职责：把标准、规则、事件和历史数据转化为建议、预警、动作和协同。

### Layer 4 — Data Asset Layer
数据资产层：

- 菜品数据
- 厨房模型
- 运营数据
- 质量历史
- 菜单知识库

职责：沉淀成餐饮行业可复用的数据资产。

## 4. 解决什么问题

餐饮行业长期存在几个核心问题：

- 同一道菜，不同厨师出品波动大
- 标准难复制，培训依赖老师傅经验
- 门店运营质量难实时感知
- 数据分散，无法形成长期资产

CaiHub 要解决的不是“看懂一张图”，而是：

```text
让一次出品有标准、有判断、有记录、有回流，并最终转化为组织资产。
```

## 5. 商业逻辑

CaiHub 的商业模式由四层组成：

### 1. 设备收入
- CaiBox 硬件部署与设备收入

### 2. 软件订阅
- 视觉质检
- 标准管理
- 运营分析
- Agent 服务

### 3. 数据资产平台
- 行业标准库
- 运营模型
- 质量知识库
- Benchmarking 数据能力

### 4. API 生态
- 向餐饮 SaaS、品牌系统、供应链系统开放能力

最终形成：

```text
餐饮行业 AI 数据平台
```

## 6. 当前仓库在整个蓝图里的位置

当前 CaiHub 仓库还不是完整的 AI Company 运行时。

当前阶段更准确的定位是：

```text
CaiHub AI Company 的后端基础平台仓
```

当前已经具备：

- 可运行的 FastAPI 后端
- 菜品、标准、出品事件三条主链路
- 最小质检裁决闭环
- 架构蓝图接口
- 数据契约与数据流治理骨架
- Agent / Skill 注册结构
- Alembic 迁移基础设施

当前尚未完全展开：

- 门店与人员领域
- 更强规则引擎
- 真正事件总线
- Agent 执行运行时
- 数据分析导出与训练闭环

## 7. 当前差异化价值

### 1. 不是只做视觉识别
识别只是输入层，不是全部价值。

### 2. 不是只做单店工具
仓库已经按平台与数据资产方向设计。

### 3. 不是把判断写死在模型里
系统明确分离：
- 识别负责观察
- 标准负责约束
- 规则负责裁决
- Agent 负责编排
- 数据资产负责复利

## 8. 一句话总结

```text
龙虾公司做的是 AI 模拟公司，而 CaiHub 做的是 AI 重构餐饮行业。
```

换成更面向业务的话：

```text
CaiHub 不是在卖一个后厨识别工具，而是在构建餐饮行业的 AI 原生运营系统。
```
