# CaiHub 版本发布说明

版本：`v0.1.0`

发布日期：`2026-03-09`

## 版本定位

这是 CaiHub 的首个基础版本。

它还不是完整的 AI Company 运行时，但已经明确对齐到下面这个方向：

```text
CaiHub 是餐饮行业的 AI 运营系统。
```

这个版本的意义，不是把所有业务一次做完，而是先搭好通往 AI Company 的平台底座：

- FastAPI 后端骨架
- 菜品、标准、出品事件三条主链路
- 最小质量裁决闭环
- Data Mesh 视角的数据契约与数据流
- Agent / Skill 注册与治理结构
- Alembic 迁移基础设施

## 本次发布包含什么

### 1. 基础服务层
- FastAPI 应用工厂
- API 路由与依赖注入
- service / repository 分层
- PostgreSQL 默认支持

### 2. 当前已落地业务对象
- 菜品 `dish`
- 菜品标准 `dish_standard`
- 出品事件 `production_event`

### 3. 最小质检闭环
- 创建出品事件
- 按菜品标准执行规则计算
- 输出质检结果
- 生成质量领域事件

### 4. AI Company 治理底座
- 数据契约目录
- Data Mesh 元数据注册
- 数据流注册
- Agent / Skill 注册表
- 系统架构蓝图接口

### 5. 数据库迁移
- Alembic 已接入
- 提供初始 schema migration

## 当前接口能力

- `GET /api/v1/health`
- `GET /api/v1/system/info`
- `GET /api/v1/system/architecture`
- `GET /api/v1/dishes`
- `POST /api/v1/dishes`
- `GET /api/v1/standards`
- `POST /api/v1/standards`
- `GET /api/v1/production/events`
- `POST /api/v1/production/events`
- `POST /api/v1/vision/dish-recognition`

## 当前适合用来做什么

- 作为 CaiHub AI Company 的后端底座继续开发
- 演示“菜品标准 + 出品事件 + 质量裁决”的基础闭环
- 验证 Agent / Skill / 数据契约 / 数据流的治理模型
- 为后续 `store`、`operator`、`rule_engine`、事件总线和 Agent runtime 铺路

## 当前还不包含什么

- 门店和人员实体建模
- 完整规则引擎 DSL
- 真实消息总线
- 报表与分析导出
- 可执行的 Agent runtime
- 硬件层完整适配

## 推荐后续版本方向

- 规则引擎升级
