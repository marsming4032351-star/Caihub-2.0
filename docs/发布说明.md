# CaiHub 首个版本发布说明

版本：`v0.1.0`

发布日期：`2026-03-09`

## 版本定位

这是 CaiHub 的第一个后端基础版本。

它的目标不是一次性做完完整业务系统，而是先搭好平台化演进所需的核心边界：

- FastAPI 后端骨架
- 数据模型与服务分层
- 菜品、标准、出品事件三条主链路
- Data Mesh 视角的数据契约与数据流
- agent / skill 注册表
- Alembic 迁移基础设施

## 本次发布包含什么

### 1. 后端服务骨架

- FastAPI 应用工厂
- API 路由与依赖注入
- service / repository 分层
- PostgreSQL 默认支持

### 2. 核心业务对象

- 菜品 `dish`
- 菜品标准 `dish_standard`
- 出品事件 `production_event`

### 3. 质量裁决最小闭环

- 创建出品事件
- 按菜品标准执行规则计算
- 输出质检结果
- 生成质量领域事件

### 4. 治理与平台骨架

- 数据契约目录
- Data Mesh 元数据注册
- 数据流注册
- agent / skill 注册表
- 系统架构蓝图接口

### 5. 数据库迁移

- Alembic 已接入
- 提供初始 schema migration

## 当前接口能力

- `GET /api/v1/health`
- `GET /api/v1/system/info`
- `GET /api/v1/system/architecture`
- `GET /api/v1/dishes`
- `POST /api/v1/dishes`
- `GET /api/v1/standards`
- `POST /api/v1/standards`
- `GET /api/v1/production/events`
- `POST /api/v1/production/events`
- `POST /api/v1/vision/dish-recognition`

## 当前适合用来做什么

- 作为 CaiHub 后端基础仓继续开发
- 作为平台化数据结构和服务边界的基线版本
- 用于演示“菜品标准 + 出品事件 + 质量裁决”的基础闭环
- 用于继续扩展 `store`、`operator`、`rule_engine`、事件总线等模块

## 当前还不包含什么

- 门店和人员实体建模
- 完整规则引擎 DSL
- 真实消息总线
- 报表与分析导出
- 可执行的 agent 运行时

## 推荐后续版本方向

### `v0.2.x`

- `store` / `operator` 领域
- 更完整的关系模型
- 规则引擎升级

### `v0.3.x`

- 事件总线
- 数据导出
- 契约兼容性测试

### `v0.4.x`

- agent / skill 执行运行时
- 更强的治理与自动化能力
