# 智能体设计

## 当前规划的智能体

- CaiHub CEO Agent
- Vision QA Agent
- Menu R&D Agent
- Store Ops Agent
- Marketing Agent
- Data Asset Agent

## 目标

- 让 Agent 理解系统蓝图、契约和数据流
- 让不同 Agent 对应真实餐饮组织中的职责单元
- 避免 Agent 绕开治理边界直接修改核心数据定义
- 让 Agent 的输出最终沉淀为可复用的数据资产

## 当前状态

- 当前仓库已经具备 Agent 注册表
- 当前仍是治理层和结构层
- 还不是完整的执行运行时
